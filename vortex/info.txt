VORTEX CLI ARCHITECTURE GUIDE
==============================

This file describes the system modules and their responsibilities.

FOLDER: vortex/core/
--------------------
- cli.py: The main REPL engine. Orchestrates all modules and runs the input loop.
- updater.py: Dynamic Git-based update system. Uses 'ls-remote' for real-time remote state analysis and 'checkout --detach' for clean tag/commit switching. Features ancestry tracking to identify tracking tracks (main/FIX) in detached states and 'fetch --prune' for accurate remote history.
- auth.py: Manages .env configuration and database credentials setup.

FOLDER: vortex/ui/
------------------
- banner.py: Logic for generating and displaying the CLI header (ASCII art).
- style.py: prompt_toolkit and rich styling definitions.
- commands.py: Descriptions of commands and mapping for the completer.
- completer.py: Custom logic for nested tab-completion with meta-descriptions.

FOLDER: vortex/ui/engine/
-------------------------
- core.py: Universal Interactive Pager (VUE). Logic for paginated tables and selection.
- README.md: Detailed manual for VORTEX UI ENGINE API.

FOLDER: vortex/config/
----------------------
- manager.py: Single source of truth for VERSION, PROJECT_ROOT, and local settings.
- __init__.py: Exposes version and config for setuptools and other modules.

FOLDER: vortex/database/
------------------------
- db.py: Unified Database UI Interface. Orchestrates core, explorer, and console modules.
- core.py: Low-level engine (ConnectionManager). Centralized query execution with dict_row.
- explorer.py: Navigation hub for database tables.
- schema.py: Module for table structure and column management.
- browser.py: Interactive data browser for row-level viewing.
- console.py: Dedicated SQL Console (REPL) with private command history.
- README.md: Comprehensive technical manual for the Database Ecosystem.

FOLDER: .vortex_data/ (Internal Data)
-------------------------------------
- history: Main CLI command history.
- sql_history: Dedicated history for SQL Console queries.
- settings.json: Local CLI persistent preferences.

FILE: vortex/registry.py
------------------------
- Centralized source of truth for absolute file paths and module names.

FILE: vortex/__init__.py
------------------------
- Package entry point. Exports 'main' to point to 'vortex.core.cli.main'.

EXTERNAL FILES (ROOT):
----------------------
- .env: Database credentials (git-ignored).
- pyproject.toml: Build configuration and dependency list.
- MODULES_GUIDE.md: Strict workflow instructions for system components.
- GEMINI.md: Mandatory engineering guide and system file map.
