# Vortex Database Ecosystem

Интерактивная модульная среда для управления PostgreSQL внутри Vortex CLI.

## 🏗 Архитектура

Модуль разделен на специализированные компоненты для обеспечения максимальной производительности и чистоты кода:

1.  **`core.py` (Engine)**: 
    - Класс `ConnectionManager`.
    - Низкоуровневые операции: соединение, транзакции, откат ошибок.
    - Централизованный метод `_execute(query, params, fetch, commit)`.
    - Сбор общей статистики БД.

2.  **`explorer.py` (Navigation)**:
    - Класс `TableExplorer`.
    - Главный хаб навигации. Список таблиц с примерным кол-вом строк.
    - Перенаправляет пользователя в Schema или Browser.

3.  **`schema.py` (Structure)**:
    - Класс `SchemaManager`.
    - Просмотр колонок, их типов, значений по умолчанию.
    - Операции над структурой: удаление и переименование колонок.

4.  **`browser.py` (Data)**:
    - Класс `DataBrowser`.
    - Просмотр первых 200 строк любой таблицы через `pager`.
    - Действия над строками (в разработке).

5.  **`console.py` (REPL)**:
    - Класс `SQLConsole`.
    - Изолированная среда в `alternate screen buffer`.
    - **Своя история**: `.vortex_sql_history`.
    - Поддержка мыши и колесика.

6.  **`db.py` (Orchestrator)**:
    - Класс `DatabaseManager`.
    - Единая точка входа для основного CLI (`cmd_db`).
    - Координирует вызовы между всеми под-модулями.

## 📜 Правила разработки (Mandates)

- **UI Isolation**: Весь визуальный вывод таблиц и списков ОБЯЗАН проходить через `vortex.ui.engine.pager`. Использование `rich.Table` напрямую запрещено (кроме специальных случаев).
- **Data Integrity**: Все выборки данных ОБЯЗАНЫ использовать `psycopg.rows.dict_row`.
- **Destructive Safety**: Любая операция, изменяющая или удаляющую данные/структуру (DROP, TRUNCATE, DELETE), требует ввода `yes` для подтверждения.
- **Transactions**: Всегда проверяйте `conn.info.transaction_status`. В случае ошибки в `core.py` реализован автоматический `rollback` перед следующим запросом.
- **Mouse Support**: Все интерактивные окна должны поддерживать мышь (колесико для прокрутки, клик для выбора).

## 📁 Служебные файлы
- `.vortex_sql_history`: История SQL-запросов (не смешивать с основной историей CLI).
- `.env`: Параметры подключения (HOST, PORT, NAME, USER, PASSWORD).
